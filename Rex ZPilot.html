Le retour d'expérience

Les avantages de la solution : NodeJS

  • Déployable partout!
  • Accessible depuis n'importe quel navigateur (moderne ;) )

Les avantages de la solution : Couchbase

  • NoSQL : adapté aux forts volumes
  • Adapté à notre besoin : Write once, Read a lot

Les avantages de la solution : ElasticSearch

  • Utilisation extensive de facets : Une requête - toutes les données
  • Gain de temps de traitement
  • Peu de métier

Les avantages de la solution : AngularsJS

  • Création d'UI simplifiée
  • Gain de temps de traitement
  • Peu de métier

Les avantages de la solution : UI Javascript

  • D3.js
    • Une librairie complète
    • Tous types de graphiques
    • De nombreuses surcouches existantes
  • elastic.js
    • Communication simplissime entre JS et ES
    • Abstraction des requêtes
  • underscore.js : librairie complète et indispensable

Les inconvénients/problèmes : Couchbase/ES

  • Documents réfléchis pour ES
  • Dupliqués entre CB et ES

Les inconvénients/problèmes : GCE

  • Communication WPilot <-> Couchbase gourmande
  • Nombre de ports ouverts sur GCE limité

Les inconvénients/problèmes : Industrialisation

  • Minification non testée au fil de l'eau
  • Conflit de dépendances entre les librairies
  • Bugs natifs des librairies

Les inconvénients/problèmes : Performances

  • Latence le jour J : Jeu presque injouable
  • Gestion du cache "à la main"

Les inconvénients de la solution : D3.js

  • Une API native complexe
  • De nombreuses surcouches existantes

Les inconvénients de la solution : UI Javascript

  • D3.js
    • Une API native complexe
    • Des effets consommateurs de ressources
  • Trop de données + Trop de graphes => lenteur pour le navigateur client
  • Beaucoup (trop) de librairies => dont certaines buggées : angularjs-nvd3-directives...
